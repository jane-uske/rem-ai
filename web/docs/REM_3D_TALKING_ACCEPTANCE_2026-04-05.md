## 执行记录

### 1. 工程检查

- `npm run typecheck`：通过
- `npm run build --prefix web`：通过
- `git diff --check`：通过

### 2. 正常回复链路

测试方式：

- 建立 WebSocket 连接
- 发送一句简短文本消息
- 观察服务端返回事件序列

结果：

- 收到 `emotion`
- 收到 `voice`
- 收到 `chat_chunk`
- 收到 `chat_end`
- 前端在本地播放 drain 后再回到 `confirmed_end`
- 期间伴随 `avatar_frame`

结论：

- 正常回复链路未被 3D 说话线程破坏
- TTS 音频输出与角色表现驱动链路兼容

### 3. 打断场景

测试方式：

- 先发送一条较长消息触发第一轮回复
- 在第一轮回复刚开始播报时发送第二条短消息
- 观察 generation 切换与中断事件

结果：

- 第一轮回复开始后收到 `voice`
- 随后收到 `interrupt (generationId: 1)`
- 第一轮 `chat_end`
- 第一轮本地播放被清队列
- 第二轮开始新的 `emotion` / `voice` / `chat_chunk` / `chat_end`

结论：

- 打断链路有效
- 旧 generation 可以被新 generation 正常抢占
- 角色说话状态具备快速收住的前提条件

### 4. 视觉验收

验收方式：

- 使用 Chromium headless 打开真实页面
- 采集待机、说话中两帧截图
- 对比角色朝向、头部姿态、眼神偏移和整体说话观感

第一次视觉检查结论：

- 角色说话中后段偏头过大
- 容易从“自然交流”滑到“明显侧过去”
- 这一点会削弱真人感

本轮修正：

- 压低说话期眼神漂移幅度
- 压低 idle / quiet 状态下的侧向漂移
- 放慢眼神切换，让回中更自然

修正后视觉结论：

- 角色在说话中仍有轻量朝向变化
- 但已回到“轻微偏头交流”的范围
- 没有再出现第一轮里那种明显大侧身观感
- 整体更接近“在看着你说话”，而不是“越说越转过去”

限制说明：

- 自动化脚本稳定抓到了待机和说话中画面
- 但“说完后完全回到 idle 的精确截图”在开发环境中不够稳定
- 因此收尾自然度本轮判定为“基本通过，建议再补一轮人工视觉复核”
