- 验收日期：2026-04-05
- 验收范围：3D 模型同步说话线程
- 验收分支：`main`

- 验收方式：构建检查 + WebSocket 联调 + 打断场景验证

## 验收目标

- 确认 TTS 播放时角色具备稳定开口表现
- 确认轻量表情、眼神、头部微动作提升真人感
- 确认停止播放与打断后角色状态能快速回落
- 确认不破坏当前实时对话主链路

## 验收结果摘要

- 总体结论：通过，可继续在 `main` 上做体验细调
- MVP 结论：通过
- 主链路兼容性：通过
- 打断体验：通过
- 非阻塞观察项：开发环境下 `/health` 命中 Next 404，不属于本线程阻塞项

## 执行记录

### 1. 工程检查

- `npm run typecheck`：通过
- `npm run build --prefix web`：通过
- `git diff --check`：通过

### 2. 正常回复链路

测试方式：

- 建立 WebSocket 连接
- 发送一句简短文本消息
- 观察服务端返回事件序列

结果：

- 收到 `interrupt`
- 收到 `emotion`
- 收到 `voice`
- 收到 `chat_chunk`
- 收到 `chat_end`
- 期间伴随 `avatar_frame`

结论：

- 正常回复链路未被 3D 说话线程破坏
- TTS 音频输出与角色表现驱动链路兼容

### 3. 打断场景

测试方式：

- 先发送一条较长消息触发第一轮回复
- 在第一轮回复刚开始播报时发送第二条短消息
- 观察 generation 切换与中断事件

结果：

- 第一轮回复开始后收到 `voice`
- 随后收到 `interrupt (generationId: 1)`
- 第一轮 `chat_end`
- 第二轮开始新的 `emotion` / `voice` / `chat_chunk` / `chat_end`

结论：

- 打断链路有效
- 旧 generation 可以被新 generation 正常抢占
- 角色说话状态具备快速收住的前提条件
